搭建hexo还有使用git的坑

发表于 2016-08-04

##安装完成后注意事项

  • 博客系统根目录下的_config设置需要注意:
    1.添加字段需要在每行进行置顶。
    2.字段后的设置的值必须与:有一个空格的距离

  • 不用每次在git命令行输入密码的方法:
    1.系统是windows的情况下可以直接在用户环境变量中添加HOME键，值为:%USERPROFILE%,进入系统的%HOME%目录（目录查询可以在git中输入echo $HOME进行查询，一般是C:\Users\用户名这样的路径），新建文件_netrc没有文件名，此为文件后缀名，输入以下文字:

    machine github.com
login username(用户名)
password \*\*\*\***(输入密码)

    2.全程git 命令创建
    在%HOME%目录下进行以下命令的输入:

    touch .git-credentials （创建文件）

vim .git-credentials （编辑文件）

https://{username}:{password}@github.com （文件里面的内容）

    接着在git终端输入以下命令来进行相关配置:

    git config --global credential.helper store

  • 下载主题以及应用:
    1.输入命令:

    git clone xxxxxx（此为主题的网址）

    2.在根目录config文件中修改theme字段为相对应字段。
    3.各个theme的配置不尽相同，需根据主题相关页面进行设置。

  • 主题应用时路径的相关设置：

由于目前的githubpage都使用了https，在下载完主题需要应用的时候，需要对各个分类的路径进行设置比如：

# When running the site in a subdirectory (e.g. domain.tld/blog), remove the leading slash (/archives -> archives)
menu:
  home: /
  #categories: /categories/
  #about: /about/
  archives: /archives/
  tags: tags/
  #sitemap: /sitemap.xml
  #commonweal: /404.html

这些都是子目录，需要在路径后面加上/，否则跳转https的时候会默认进行http的链接导致博客的无法访问。
